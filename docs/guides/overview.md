# PAGE 개요

> English: [overview_en.md](https://monkshark.github.io/page-ide/#guides/overview_en.md)

> 다언어 데스크톱 IDE. 이름은 핵심 가치 네 개의 머리글자 — Pair · Atlas · Glass · Echo.

PAGE는 한 페이지 안에서 네 가지 차원을 동시에 보는 것을 목표로 한다. 코드(텍스트), 코드 그래프(공간), 작업 시간축(시간), AI 동반자(대화). 기존 IDE가 이 중 하나만 잘 푸는 동안, PAGE는 이 넷을 한 화면 안에 통합하려 한다.

---

## 현재 구현 상태

이 개요는 지향점 네 가지를 함께 설명한다. 그중 지금 동작하는 것과 아직 계획 단계인 것을 먼저 구분한다.

| 차원 | 상태 |
| --- | --- |
| 에디터 · LSP (기반) | 동작 — 다언어 편집, 진단, 자동완성 |
| Atlas | 동작 — import · 호출 · 모듈 그래프 |
| Glass | 부분 — 디자인 토큰, 글래스 표면, 다크/라이트 테마 |
| Pair | 계획 — 아직 코드 없음 |
| Echo | 계획 — 아직 코드 없음 |

아래 네 절은 완성된 모습을 함께 설명하므로, 각 절 첫 줄에 현재 상태를 표시한다.

---

## 핵심 가치 네 가지

### Pair — AI 동반자

> 상태: 계획 단계. 아래는 목표 동작이며 아직 구현되지 않았다.

곁에서 코드를 함께 보는 모드. 명령을 받고 실행하는 외주형이 아니라, 사용자의 작업을 관찰하고 짧게 거드는 동행형을 지향한다.

- 관찰자 — 사용자가 짜는 코드를 보고 자발적으로 짧게 코멘트.
- 대화 — 채팅 기반 질문/답변, 코드 컨텍스트 자동 첨부.
- 에이전트 — 도구 호출로 파일 수정, 명령 실행. 모든 변경은 사용자 승인 필요.
- 튜터 — 학습자용. 코드의 의도를 묻고 대안을 제시한다.

### Atlas — 코드 그래프 (공간)

> 상태: 동작. import · 호출 · 모듈 그래프 뷰를 제공한다.

파일 트리는 디렉터리 구조일 뿐이다. Atlas는 모듈, 함수, 의존성을 노드와 엣지로 시각화한다.

- 노드: 파일 / 주요 심볼
- 엣지: import, 호출, 상속
- 줌·팬, 노드 클릭으로 파일 오픈, 호버 미리보기
- "현재 파일에서 도달 가능한 노드만 보기" 필터

### Glass — UI 미감

> 상태: 부분 구현. 디자인 토큰과 글래스 표면, 다크/라이트 테마 전환이 있다. 포커스 모드 등은 예정.

매일 8시간 보는 화면이라면 보고 싶은 화면이어야 한다. Glass는 글래스모피즘 기반 디자인 시스템이다.

- 다크 모드 기본, 라이트 모드 전환
- 반투명 패널, 부드러운 모션, 네온 액센트
- 포커스 모드 (현재 파일 외 흐림)
- Compose Multiplatform Material 3 위에 디자인 토큰 v1

### Echo — 키스트로크 타임라인 (시간)

> 상태: 계획 단계. 아직 구현되지 않았다.

Git 커밋은 결과 스냅샷이지, 작업 흐름이 아니다. Echo는 키 입력을 로컬 SQLite에 저장해서 작업 시간축을 그대로 되감을 수 있게 한다.

- 디바운스된 키 입력 자동 레코딩 (오프라인, 로컬 DB)
- 하단 타임라인 바, 스크럽으로 시점 이동 (read-only 뷰)
- "이 시점으로 복원" 버튼
- 보존 기간 기본 30일, 커스터마이즈 가능

## 만들지 않을 것

스코프 보호는 컨셉의 일부다.

- 모바일/웹 버전 — 데스크톱 우선
- 자체 언어 서버 — LSP 클라이언트만
- 자체 디버거 엔진 — 추후 DAP 사용
- 협업 편집 (Live Share 류) — 후순위
- 마켓플레이스/플러그인 SDK — v2.0 이후

---

## 단계 구성

네 단계로 나누어 단계마다 배포 가능한 결과물을 남긴다.

1. 기반 — 예쁜 다언어 에디터.
2. Atlas + Git — 코드 그래프와 버전 관리 통합.
3. Echo — 키스트로크 타임라인.
4. Pair — AI 동반자 (멀티 프로바이더).
